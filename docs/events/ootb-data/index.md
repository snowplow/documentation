Snowplow has provided these out-of-the-box [event](/docs/fundamentals/events/index.md) and [entity](/docs/fundamentals/entities/index.md) types so you can get started quickly with tracking user data.

This table shows the available events or out-of-the-box data grouped by category.

| Event category                                                                                     | Description                                              |
| -------------------------------------------------------------------------------------------------- | -------------------------------------------------------- |
| [Application errors](/docs/events/ootb-data/app-error-events/index.md)                             | Track errors in your application                         |
| [Application and tracker information](/docs/events/ootb-data/app-information/index.md)             | Data about the application and tracker                   |
| [Application performance](/docs/events/ootb-data/app-performance/index.md)                         | Track application performance and timings                |
| [Campaigns and ads](/docs/events/ootb-data/campaigns-and-ads/index.md)                             | Track marketing campaigns and ads on web                 |
| [Consent](/docs/events/ootb-data/consent-events/index.md)                                          | Track user marketing consent on web                      |
| [Device and browser information](/docs/events/ootb-data/device-and-browser/index.md)               | Data about the user's device and browser                 |
| [Ecommerce](/docs/events/ootb-data/ecommerce-events/index.md)                                      | Track ecommerce activity                                 |
| [Geolocation](/docs/events/ootb-data/geolocation/index.md)                                         | Data about the user's location                           |
| [Referrers, links, and cross-navigation](/docs/events/ootb-data/links-and-referrers/index.md)      | How users arrive at or navigate through your application |
| [Media](/docs/events/ootb-data/media-events/index.md)                                              | Track video or audio interactions                        |
| [Mobile application lifecycle](/docs/events/ootb-data/mobile-lifecycle-events/index.md)            | Track mobile application installs and lifecycle          |
| [Page and screen engagement](/docs/events/ootb-data/page-activity-tracking/index.md)               | Data about user engagement                               |
| [Page and screen views](/docs/events/ootb-data/page-and-screen-view-events/index.md)               | Track page views on web and screen views on mobile       |
| [Page elements](/docs/events/ootb-data/page-elements/index.md)                                     | Track button clicks, forms, and element visibility       |
| [Site search](/docs/events/ootb-data/site-search/index.md)                                         | Track internal site search queries                       |
| [Social media interactions](/docs/events/ootb-data/social-media/index.md)                          | Track likes, shares, and social widget interactions      |
| [Third-party integrations](/docs/events/ootb-data/third-party-integrations/index.md)               | Capture data from Optimizely, GA cookies, and more       |
| [User and session identification](/docs/events/ootb-data/user-and-session-identification/index.md) | Identifiers for users and sessions                       |
| [VisionOS and SwiftUI](/docs/events/ootb-data/visionos-swiftui/index.md)                           | Track user behavior in SwiftUI and VisionOS              |
